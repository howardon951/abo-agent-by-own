# 頁面 Wireframe 與資訊架構

## 1. Merchant Dashboard

目的：

- 商家登入後第一眼掌握系統是否正常運作

```text
+----------------------------------------------------------------------------------+
| Top Nav: Logo | Dashboard | Agent | Scenarios | Knowledge | LINE | Conversations |
+----------------------------------------------------------------------------------+
| Tenant Summary                                                                   |
| Plan: Basic | LINE: Connected | Agent: Active | KB Docs: 6 | Open Handoffs: 2    |
+----------------------------------------------------------------------------------+
| Cards                                                                             |
| [Messages Today] [Handoffs Today] [Errors] [Est. Cost]                           |
+----------------------------------------------------------------------------------+
| Recent Conversations                                                             |
| - LINE User A | bot_active   | 請問營業到幾點？                                  |
| - LINE User B | human_active | 我想退貨                                         |
+----------------------------------------------------------------------------------+
| System Alerts                                                                    |
| - 1 document failed to process                                                   |
| - LINE webhook verified                                                          |
+----------------------------------------------------------------------------------+
```

## 2. Agent Settings

目的：

- 以表單方式管理品牌規則，不讓商家直接寫失控 prompt

```text
+---------------------------------------------------------------+
| Agent Settings                                                |
+---------------------------------------------------------------+
| Agent Name        [ Main Agent                             ]  |
| Brand Name        [ Abo Coffee                             ]  |
| Brand Tone        [ 親切、簡短、專業                        ]  |
| Forbidden Topics  [ 醫療建議, 法律意見                     ]  |
| Fallback Policy   [ 若不確定，請保守回覆並引導真人         ]  |
| Business Hours    [ JSON/Form later ]                        |
|                                                               |
| [ Save ]                                                     |
+---------------------------------------------------------------+
```

## 3. Scenario Settings

目的：

- 管理不同情境規則與路由關鍵字

```text
+----------------------------------------------------------------------------------+
| Scenario List                                                                    |
+----------------------------------------------------------------------------------+
| [一般 FAQ]   Enabled | Keywords: -                                               |
| [售前問答]   Enabled | Keywords: 價格, 費用, 方案                                 |
| [售後問答]   Enabled | Keywords: 退貨, 退款, 維修                                 |
| [門市資訊]   Enabled | Keywords: 地址, 營業時間, 停車                            |
+----------------------------------------------------------------------------------+
| Selected Scenario: 售前問答                                                      |
| Objective      [ 回答購買前問題 ]                                                |
| Style          [ 清楚、避免誤導 ]                                                |
| Keywords       [ 價格, 費用, 方案 ]                                              |
| [ Save ]                                                                         |
+----------------------------------------------------------------------------------+
```

## 4. Knowledge Base

目的：

- 管理 FAQ / PDF / URL 文件與處理狀態

```text
+----------------------------------------------------------------------------------+
| Knowledge Base                                                                    |
+----------------------------------------------------------------------------------+
| [ Add FAQ ] [ Upload PDF ] [ Add URL ]                                           |
+----------------------------------------------------------------------------------+
| Documents                                                                         |
| 常見問題          | FAQ | Ready      | 2026-04-01 12:00                          |

| 產品手冊.pdf      | PDF | Processing | -                                         |
+----------------------------------------------------------------------------------+
| Document Detail                                                                   |
| Title: 門市資訊頁                                                                 |

| Status: Ready                                                                     |

+----------------------------------------------------------------------------------+
```

## 6. Conversations List

目的：

- 讓商家快速找到需要人工介入或異常的對話

```text
+----------------------------------------------------------------------------------+
| Filters: [ All Status ] [ Search Contact ] [ Date Range ]                        |
+----------------------------------------------------------------------------------+
| Contact        | Status        | Last Message              | Updated             |
| LINE User A    | bot_active    | 今天營業到幾點？          | 13:00               |
| LINE User B    | human_active  | 我要退貨                  | 13:05               |
| LINE User C    | closed        | 謝謝                      | 12:20               |
+----------------------------------------------------------------------------------+
```

## 7. Conversation Detail

目的：

- 查看完整訊息與 bot/human 狀態，並可恢復 AI

```text
+----------------------------------------------------------------------------------+
| Conversation: LINE User B                                                         |
| Status: human_active     [ Resume AI ]                                            |
+----------------------------------------------------------------------------------+
| User: 我想退貨                                                                     |
| Bot : 已轉由真人協助，請稍候                                                       |
| User: 請盡快                                                                       |
+----------------------------------------------------------------------------------+
| Side Panel                                                                         |
| Contact ID: xxx                                                                    |
| Scenario: post_sales                                                               |
| Last Retrieval: 3 chunks                                                           |
| Last LLM Call: skipped (human_active)                                              |
+----------------------------------------------------------------------------------+
```

## 8. Playground

目的：

- 後台驗證 prompt 與知識庫品質

```text
+----------------------------------------------------------------------------------+
| Playground                                                                        |
+----------------------------------------------------------------------------------+
| Input                                                                             |
| [ 今天營業到幾點？                                                      ] [ Run ] |
+----------------------------------------------------------------------------------+
| Output                                                                            |
| 我們每日營業時間為 10:00 到 20:00。                                               |
+----------------------------------------------------------------------------------+
| Debug Info                                                                         |
| Scenario: 門市資訊                                                                 |
| Retrieved Chunks                                                                   |
| - FAQ / score 0.92 / 營業時間為每日 10:00-20:00                                   |
| - 門市資訊頁 / score 0.87 / 地址與營業資訊                                        |
+----------------------------------------------------------------------------------+
```

## 11. MVP 導覽結構

Merchant：

- Dashboard
- Agent
- Scenarios
- Knowledge
- LINE
- Conversations
- Playground
- Settings

Platform：

- Platform Dashboard
- Tenants
- Logs
- Usage

## 12. 刻意不設計的畫面

以下畫面先不做：

- 多 agent 切換器
- 內建人工客服回覆介面
- 複雜權限管理頁
- 計費付款頁
- 多通路管理頁
